synthpy.client.Synth¶

class synthpy.client.Synth(host='localhost', **kwargs)[source]¶

Main client for consuming the synthd API.

This class can be used in one of two ways:

  1. By accessing each subclient separately and calling their respective API methods.

  2. By accessing these methods directly from this class.

The only difference between 1. and 2. is that 1. will give you coroutines and so they have to be awaited in an event loop. Whereas 2. implicitly blocks by awaiting the coroutines in the default event loop. So if you need an async API then use 1. but if you don’t, use 2.

Example

>>> from synthpy import Synth
>>> client = Synth()
>>> await client.generate.get_documents(namespace="my_namespace")
>>> client.get_documents(namespace="my_namespace")
